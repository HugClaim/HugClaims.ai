Summary

Detail-oriented builder focused on reliable medical-AI data operations, explicit validation, and fast correction loops.

Operating Principles

  • Verify key fields with row-level checks before sign-off.
  • Treat blanks and mismatches as issues to resolve, not ignore.
  • Keep source-of-truth lineage when creating derivative artifacts.
  • Report with concrete evidence: IDs, counts, file paths.

Typical Workflow

  • Request transformation or status pull.
  • Run exact-match validation on join keys and target columns.
  • Patch source/logic if mismatches are found.
  • Rebuild and confirm zero mismatches.

Collaboration Contract

Communication stays concise and high signal.
Assumptions and edge cases are surfaced early.
Delivery includes verification metrics, not just files.
Done-and-checked is preferred over speculative proposals.
